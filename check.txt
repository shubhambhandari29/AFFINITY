Entra ID Group Authorization for FastAPI Backend
(F5 APM + React + Azure App Service + Microsoft Graph)
1. Overview
This document describes the design and implementation for retrieving Microsoft Entra ID (Azure AD) group memberships for authenticated users in our application.
The application uses F5 APM for authentication, and the backend determines the user's role based on Entra ID group membership.
End-to-End Flow
User accesses the application URL protected by F5 APM.
F5 authenticates the user using enterprise authentication.
After authentication, F5 injects the User ID in an HTTP header.
The React UI receives the request and forwards the User ID to the backend.
The FastAPI backend queries Microsoft Graph API.
Microsoft Graph returns the list of AD groups the user belongs to.
Backend filters only the groups relevant to the application.
Backend determines the application role.
Role information is returned to React UI.
Example Role Mapping
AD Group	Application Role
SAC_Admin	Admin
SAC_Director	Director
SAC_Underwriter	Underwriter
2. Methods to Access Microsoft Graph
Two possible approaches exist for the backend to authenticate with Microsoft Graph.
1️⃣ App Registration (Client ID + Secret)
2️⃣ Managed Identity (Recommended)
3. Method 1 – Entra App Registration
This method uses an application identity registered in Microsoft Entra ID.
The FastAPI backend authenticates using Client Credentials OAuth flow.
Admin Setup Steps
Go to Microsoft Entra Admin Center
Navigate to:
App Registrations → New Registration
Create a new application.
Example name:
SAC-FastAPI-Backend
After creation note the following:
Field	Purpose
Application (Client) ID	Used by backend to request token
Directory (Tenant) ID	Azure tenant identifier
Create Client Secret
Navigate to:
Certificates & Secrets → New Client Secret
Save the generated secret.
⚠️ Store the secret securely in Azure Key Vault.
Grant Microsoft Graph Permissions
Navigate to:
API Permissions → Add Permission
Select:
Microsoft Graph
Then choose:
Application Permissions
Add:
Directory.Read.All
Admin Consent
After adding permission:
Grant Admin Consent
Without admin consent the Graph calls will fail.
Backend Implementation Steps
Backend stores:
Tenant ID
Client ID
Client Secret
Then performs the OAuth Client Credentials Flow.
Request token:
POST https://login.microsoftonline.com/{tenant-id}/oauth2/v2.0/token
Scope:
https://graph.microsoft.com/.default
Use returned access token to call Microsoft Graph.
4. Method 2 – Managed Identity (Recommended)
This is the preferred approach for this project.
Because the FastAPI backend is running in Azure App Service, it can use System Assigned Managed Identity.
Managed Identity allows the application to authenticate with Microsoft services without storing any secrets.
Advantages
No client secrets
No secret rotation
More secure
Azure automatically manages identity
Simpler operational maintenance
Admin Setup Steps
Step 1 — Verify Managed Identity
In Azure Portal:
App Service → Identity → System Assigned → Enabled
When enabled, Azure automatically creates a service principal in Entra ID.
Step 2 — Locate the Identity
In Microsoft Entra Admin Center:
Enterprise Applications
Search for the App Service name.
This is the service principal for the managed identity.
Step 3 — Grant Graph Permissions
Navigate to:
Enterprise Application
→ Permissions
→ Add Permission
Add:
Microsoft Graph
→ Application Permissions
→ Directory.Read.All
Step 4 — Grant Admin Consent
Click:
Grant Admin Consent
Without admin consent Graph will return 403 errors.
5. Microsoft Graph API for Group Retrieval
Since our system uses group names, we must retrieve group objects instead of just IDs.
Recommended API:
GET /users/{id}/transitiveMemberOf/microsoft.graph.group?$select=id,displayName
Example:
GET https://graph.microsoft.com/v1.0/users/{user}/transitiveMemberOf/microsoft.graph.group?$select=id,displayName
Example Graph Response
{
  "value": [
    {
      "id": "7c92d21f-xxxx",
      "displayName": "SAC_Admin"
    },
    {
      "id": "8b21c111-xxxx",
      "displayName": "Finance_ReadOnly"
    }
  ]
}
6. Backend Role Mapping Logic
Backend receives list of groups.
Example Python logic:
if "SAC_Admin" in groups:
    role = "admin"

elif "SAC_Director" in groups:
    role = "director"

elif "SAC_Underwriter" in groups:
    role = "underwriter"
Backend returns resolved role to UI.
Example response:
{
  "user": "john.doe@company.com",
  "role": "admin"
}

8. Backend Implementation Steps
FastAPI backend will perform:
Step 1 — Receive User ID
React sends the user identifier.
Example:
POST /auth/login
Header:
X-User-ID: john.doe@company.com
Step 2 — Acquire Graph Token
Using Managed Identity:
ManagedIdentityCredential
Scope:
https://graph.microsoft.com/.default
Step 3 — Query Microsoft Graph
Call:
GET /users/{user}/transitiveMemberOf
Step 4 — Extract Groups
Retrieve:
displayName
from response.
Step 5 — Determine Role
Match group names against configured role groups.
Step 6 — Return Role
Send role information back to React.
